Explanation of the Pantoum

Description of the Form

A pantoum is a poem form that heavily relies on repetition. With each new stanza incorporating lines from the previous, the pantoum is a form that requires dillegence, but with my pantoum generator, you can very quickly make a first draft.

Explanation of the Form

Explaining the form of the pantoum is tricky but once understood its beauty can be appreciated. Let's make a test stanza that will show how the form works. First we need a new stanza:

Here's a first line

Here's a second

Let's add in a third

And after, a fourth

The color coding will help with following what happens after this stanza. Now let's continue the form into a second stanza.

Here's a second

This is the first new line

And after, a fourth

This is the other new line

Lines 2 and 4 of the first stanza are now lines 1 and 3 in stanza 2. This pattern of the even lines of the previous stanza becoming the odd lines of the new stanza continues until the final stanza. Here's an example final stanza if we made a 3 stanza pantoum with the previous two stanzas.

This is the first new line

Here's a first line

This is the other new line

Let's add in a third

As you can see we end up using the two odd lines of the first stanza in order to finish the form, making them the even lines and ending with the first lines of the poem. Now here's an actual poem example of a pantoum I wrote, that this generator could help draft.

Functional Overview

Elaborating the Problem being solved

Now recognizing the pattern that emerges from a pantoum, It was very simple to divide the work into reasonable sections. These sections are just individual functions.


def first_line_input()

This function serves the purpose of starting the pantoum and the automation of the ordering of lines. There's a general purpose function for the rest of the poem however due to the ending it is necessary to handle the first line differently.


def last_line()

This function serves the purpose of completing the pantoum using data saved from the def first_line_input() function. Due to the way that the last stanza is constructed, this too could not be placed in the general purpose function without using for loops.


def middle_line()

This is the general purpose code to generate the stanzas for the form. This function automatically sorts the lines inputed properly and adds them to the outputted.


def generate_pantoum()

This function runs all of the previous functions in the correct order to provide the return the output of the desired poem


The Method

The main trick to solving the problem of ordering the lines is that as long as we know the lines from the previous stanza that will go on to be new lines in the next stanza, we then know that the poet will only be writing lines 2 and 4 of the new stanza. This is to say that given a stanza [a, b, c d] we already know that the next stanza will be at least [b, _, d, _] where the underscores are new inputs or the lines from the first stanza [w, x, y, z] where the result would be [b, w, d, y].

This effectively means we only need a few temporary variables to keep track of the first lines of the poem, and a few such variables for the previous two lines of the previous stanza.

Key Functionality

poem += next_line+ linebreak + linebreak
hold_old_line2 = next_line

This is the part where we're using temporary variables to cache the previous values while adding them on to the poem. This means there's no need for counting beyond a simple for loop.

Time Complexity

In order to complete this process, the time complexity is O(n) where n is the number of lines long the stanza is.The memory complexity is O(1) as the only memory usage is for the first stanza and for caching and immediately replacing the used memory with new data.

The Code

line1 = ''
line2 = ''
line3 = ''
line4 = ''
hold_old_line1 = ''
hold_old_line2 = ''
poem = ''
linebreak = '\n'
def first_line_input():
global line1, line2, line3, line4, hold_old_line1, hold_old_line2, poem
print("Enter the four initial lines of the poem:")
#getting values from input
a = input()
b = input()
c = input()
d = input()
#setting them for the poem
line1 = a
line2 = b
line3 = c
line4 = d
#preping for the next stanza
hold_old_line1 = b
hold_old_line2 = d
poem += a + linebreak + b + linebreak + c + linebreak + d + linebreak
return
def last_line():
global poem, hold_old_line1, hold_old_line2
poem += hold_old_line1 + linebreak + line3 + linebreak + hold_old_line2 + linebreak + line1
return
def middle_line():
#prep line 1
global poem, hold_old_line1, hold_old_line2
poem += hold_old_line1 + linebreak
next_line = input()
#set line 2 and 3
poem += next_line + linebreak + hold_old_line2 + linebreak
hold_old_line1 = next_line #prep for next stanza
next_line = input()
#set line 4
poem += next_line+ linebreak + linebreak
hold_old_line2 = next_line #prep for next stanza
print ('this is the poem so far')
print (poem)
return
def generate_pantoum():
global poem
first_line_input()
lines = [line1, line2, line3, line4]
print(lines)
# Input the number of stanzas for the poem
num_stanzas = int(input("Enter the number of stanzas you want in the poem: "))
for i in range(num_stanzas - 1):
middle_line()
last_line()
return poem
print(generate_pantoum())

Future Plans and Critique

Critiques

The first critique I would have for this design is that it is only good at making drafts as you cannot edit previous data. Also only outputting to console isn't very user friendly barring that the user is already very much a technical person. At this point, I have yet to make an interface for this project, but I wish to do so sometime in the near future. Another thing is that there are variables that aren't used to their full potential. You also must go in one direction in order to creat this poem.

Plans

  • Add interface
  • Allow line by line editing
  • .txt and .doc imports
  • .txt and other file exports
  • Bigger overall program integration

Adding all of this extra functionality will require some heavy additions to the program and handling of cases that I have not accounted for, but would result in a better experience for the poets.
